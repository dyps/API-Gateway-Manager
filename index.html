🔒 Armazenamento local

Os arquivos enviados aqui não são transmitidos para nenhum servidor. Eles são lidos diretamente pelo seu navegador e salvos no IndexedDB da sua máquina.

  • Nenhum dado é enviado para a internet.
  • Os dados ficam disponíveis apenas neste navegador e neste dispositivo.
  • Limpar os dados do navegador remove os arquivos salvos.

🌍 JSON de Ambientes

Array de objetos descrevendo os ambientes disponíveis (dev, homologação, produção, etc.). Carregue este primeiro — ele habilita a troca de ambiente e o preenchimento automático dos valores ao importar outros JSONs.

  • Deve ser um array JSON ([ ... ]).
  • Cada item representa um ambiente com seus valores específicos de host, nlb, connectionId, authorizerUri, etc.
  • Com ele carregado, ao importar o JSON de Grupos ou o do API Gateway, os valores são substituídos automaticamente para o ambiente selecionado.
  • Consulte o arquivo environments.example.json para ver o formato esperado.
⬇ Baixar environments.example.json

🗂️ JSON de Grupos de Paths

Agrupa os paths do API Gateway em grupos nomeados para facilitar a gestão. Carregue antes do JSON do API Gateway para que os paths sejam exibidos organizados por grupo.

  • Deve ser um objeto onde cada chave é o nome do grupo.
  • O valor de cada grupo é um objeto com os paths (/meu/path) e suas configurações completas.
  • Com o JSON de Ambientes carregado, os valores de ambiente dentro do arquivo são substituídos automaticamente.
  • Sem o JSON do API Gateway, os grupos são exibidos mas sem status de validação (sem badges de divergência).
  • Consulte o arquivo groupPaths.example.json para ver o formato esperado.
  • Exemplo: { "MeuGrupo": { "/api/recurso": { ... } } }
⬇ Baixar groupPaths.example.json

📄 JSON do API Gateway

Arquivo de configuração exportado diretamente do AWS API Gateway (formato Swagger/OpenAPI 2.0). É o JSON principal — habilita a edição de paths, validação dos grupos e o download do JSON final.

Como exportar pelo Console AWS

  1. Acesse o AWS Console → serviço API Gateway.
  2. Selecione sua REST API.
  3. No menu lateral, clique em Stages.
  4. Selecione o stage desejado (ex: dev, prod).
  5. Clique em Stage actions → Export.
  6. No popup de exportação, selecione:
    • API specification type: Swagger
    • Format: JSON
    • Extensions: Export with API Gateway extensions
  7. Clique em Export API e salve o arquivo.
  • Deve ser um objeto JSON com a chave paths contendo os endpoints.
  • Com o JSON de Ambientes carregado, o ambiente é detectado automaticamente pelo conteúdo do arquivo.
  • Com o JSON de Grupos carregado, os paths são exibidos organizados por grupo e com badges de status.
  • Campos esperados na raiz: host, paths, securityDefinitions, etc.
